# Implementation Patterns

This document is the **canonical reference for how code is written in this project**. Every @engineer agent must read this before writing any file. These patterns are non-negotiable — deviation requires a PR updating this document first.

---

## 1. `lib/` Directory Structure

```
app_build/
  lib/
    supabase/
      client.ts        ← browser Supabase client (Client Components only)
      server.ts        ← server Supabase client (Server Components + API routes)
    db/
      types.ts         ← Kysely DatabaseSchema type (auto-generated by kysely-codegen)
      index.ts         ← Kysely instance with PostgresDialect
    auth.ts            ← requireAuth(), requireAdmin(), AuthError
    constants.ts       ← Shared constants (roles, statuses, error codes, config)
    services/
      rfp/             ← RFP lifecycle engine + state machine
      discovery/       ← 70/30 ranking, DQS calculation, PostGIS search
      subscription/    ← Subscription lifecycle, credits, trial lock
      moderation/      ← Sightengine content moderation pipeline
      notifications/   ← Event-to-notification mapping, inbox, preferences
      roles/           ← Product seller, contractor, equipment dealer validators
    utils/
      validation.ts    ← GSTIN, PAN, phone regex validators
      ranking.ts       ← 70/30 DQS formula (pure function, testable)
  middleware.ts        ← Next.js auth guard (protects routes, redirects to signin)
```

## 2. Supabase Clients

### Browser client — Client Components only
```typescript
// @witness [SPEC-ID]
// lib/supabase/client.ts
import { createBrowserClient } from '@supabase/ssr'

export function createClient() {
  return createBrowserClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
  )
}
```

### Server client — Server Components and API Routes
```typescript
// @witness [SPEC-ID]
// lib/supabase/server.ts
import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'

export function createClient() {
  const cookieStore = cookies()
  return createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() { return cookieStore.getAll() },
        setAll(cookiesToSet) {
          cookiesToSet.forEach(({ name, value, options }) =>
            cookieStore.set(name, value, options)
          )
        },
      },
    }
  )
}
```

### Middleware client — session refresh
```typescript
// @witness [SPEC-ID]
// lib/supabase/middleware.ts
import { createServerClient } from '@supabase/ssr'
import { NextResponse, type NextRequest } from 'next/server'

export async function updateSession(request: NextRequest) {
  let supabaseResponse = NextResponse.next({ request })
  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() { return request.cookies.getAll() },
        setAll(cookiesToSet) {
          cookiesToSet.forEach(({ name, value }) => request.cookies.set(name, value))
          supabaseResponse = NextResponse.next({ request })
          cookiesToSet.forEach(({ name, value, options }) =>
            supabaseResponse.cookies.set(name, value, options)
          )
        },
      },
    }
  )
  await supabase.auth.getUser()
  return supabaseResponse
}
```

### When to use which client

| Context | Client |
|---------|--------|
| `app/page.tsx` (Server Component) | `lib/supabase/server.ts` |
| `app/api/*/route.ts` (API Route) | `lib/supabase/server.ts` |
| `'use client'` component | `lib/supabase/client.ts` |
| `middleware.ts` | `lib/supabase/middleware.ts` |

```typescript
// after running: supabase db push

```

### Kysely query patterns
```typescript
// SELECT — non-PII fields only
const profile = await db
  .selectFrom('profiles')
  .select(['id', 'display_name', 'persona_type', 'city', 'state', 'dqs_score'])
  // NEVER select: phone_primary, phone_secondary, email_business
  .where('id', '=', profileId)
  .where('deleted_at', 'is', null)
  .executeTakeFirstOrThrow()

// INSERT
await db.insertInto('connections')
  .values({ requester_id: userId, target_id: targetId, status: 'REQUESTED' })
  .execute()

// UPDATE
await db.updateTable('profiles')
  .set({ handshake_credits: db.fn.sql`handshake_credits - 1` })
  .where('id', '=', userId)
  .execute()
```

### Soft-Delete Pattern
All soft-deletable tables use `deleted_at TIMESTAMPTZ` as the standard column:
- Set `deleted_at = NOW()` to soft-delete (never use `is_deleted` booleans).
- Query with `.where('deleted_at', 'is', null)` to exclude deleted records.
- Cascading: when a profile is soft-deleted, related OPEN RFPs are auto-CANCELLED.

## 4. PII Access — Always via `get_visible_contact_info()`

Never query `phone_primary`, `phone_secondary`, or `email_business` directly. Always:

```typescript
// @witness [HD-001]
const supabase = createClient()  // server client
const { data: contact, error } = await supabase.rpc('get_visible_contact_info', {
  p_viewer_id: currentUserId,
  p_target_id: targetProfileId,
})
// contact: { phone_primary, phone_secondary, email_business, is_masked }
```

If `contact.is_masked === true`, show the masked UI state. Never fall back to querying PII directly.

## 6. Atomic Handshake Acceptance

Never chain multiple DB calls for the accept transition. Use a single RPC:

```typescript
// @witness [HD-001]
const { error } = await supabase.rpc('accept_handshake', { p_connection_id: connectionId })
// This RPC atomically:
//   1. UPDATE connections SET status='ACCEPTED' WHERE id=p_connection_id
//   2. INSERT INTO address_book (both directions)
//   3. INSERT INTO unmasking_audit
// All in one transaction — defined in supabase/migrations/
```

## 7. Auth Integration

**Authentication uses cookie-based sessions via Supabase SSR** — NOT Bearer tokens.
**NEVER accept user IDs from request bodies.** Always extract the authenticated user from the Supabase session.

```typescript
// @witness [ID-001]
// lib/auth.ts
import { createServerClient } from '@/lib/supabase/server'

export class AuthError extends Error {
  constructor(
    public code: string,
    public status: number,
    message: string
  ) {
    super(message)
    this.name = 'AuthError'
  }
}

export async function requireAuth(request: Request) {
  const supabase = createServerClient()
  const { data: { session }, error } = await supabase.auth.getSession()
  
  if (error || !session) {
    throw new AuthError('AUTH_MISSING', 401, 'Authentication required')
  }
  
  return {
    id: session.user.id,
    email: session.user.email,
    role: session.user.role ?? 'user',
  }
}

export async function requireAdmin(request: Request) {
  const user = await requireAuth(request)
  if (user.role !== 'super_admin') {
    throw new AuthError('AUTH_INSUFFICIENT_ROLE', 403, 'Admin access required')
  }
  return user
}
```

### API Route Pattern

```typescript
// @witness [RFP-001]
import { requireAuth, AuthError } from '@/lib/auth'
import { rfpService } from '@/lib/services/rfp'

export async function POST(request: Request) {
  try {
    const user = await requireAuth(request)
    const body = await request.json()
    
    const rfp = await rfpService.create({
      ...body,
      requester_id: user.id, // Inject authenticated user ID
    })
    
    return NextResponse.json({ success: true, data: rfp }, { status: 201 })
  } catch (error) {
    if (error instanceof AuthError) {
      return NextResponse.json(
        { success: false, error: { code: error.code, message: error.message } },
        { status: error.status }
      )
    }
    return NextResponse.json(
      { success: false, error: { code: 'INTERNAL_ERROR', message: 'Internal server error' } },
      { status: 500 }
    )
  }
}
```

### Security Rules
1. **NEVER** accept `user_id`, `requester_id`, `responder_id` from request body
2. **ALWAYS** use `requireAuth()` to extract the authenticated user
3. **ALWAYS** inject `user.id` into service calls
4. **ALWAYS** catch `AuthError` and return proper error response
5. **Cookie-based sessions only** — Supabase SSR manages auth cookies; no Bearer tokens

## 8. API Route Shape

All responses follow the global contract from `docs/api/API_CONTRACT.md`:

### Success Response
```typescript
return NextResponse.json({
  success: true,
  data: { ... },
  meta: { page, page_size, total_count, total_pages }
})
```

### Pagination Pattern
All list endpoints MUST return paginated responses:
```typescript
{
  success: true,
  data: {
    items: [...]
  },
  meta: {
    page: 1,
    page_size: 20,
    total_count: 150,
    total_pages: 8
  }
}
```

### Error Response
```typescript
return NextResponse.json(
  {
    success: false,
    error: {
      code: 'ERROR_CODE',
      message: 'Human-readable message',
      details: { ... }  // optional, for debugging
    }
  },
  { status: 4xx | 5xx }
)
```

Standard error codes:
| Code | HTTP | Meaning |
|------|------|---------|
| `AUTH_MISSING` | 401 | No valid session |
| `AUTH_INSUFFICIENT_ROLE` | 403 | Valid session but no permission |
| `RESOURCE_NOT_FOUND` | 404 | Resource doesn't exist |
| `HANDSHAKE_INITIATE_INSUFFICIENT_CREDITS` | 402 | `handshake_credits < 1` |
| `HANDSHAKE_INITIATE_SUBSCRIPTION_LOCKED` | 403 | `subscription_status = 'hard_locked'` |
| `HANDSHAKE_INITIATE_ALREADY_CONNECTED` | 409 | Connection already ACCEPTED or BLOCKED |
| `VALIDATION_FAILED` | 400 | Input failed validation |

## 9. Input Validation Patterns

Use these server-side validators from `lib/utils/validation.ts`:

```typescript
// @witness [ID-001]
export const GSTIN_REGEX = /^[0-9]{2}[A-Z]{5}[0-9]{4}[A-Z]{1}[1-9A-Z]{1}Z[0-9A-Z]{1}$/
export const PAN_REGEX   = /^[A-Z]{5}[0-9]{4}[A-Z]{1}$/
export const PHONE_REGEX = /^\+91[0-9]{10}$/

export function isValidGSTIN(value: string): boolean {
  return GSTIN_REGEX.test(value)
}
export function isValidPAN(value: string): boolean {
  return PAN_REGEX.test(value)
}
export function isValidPhone(value: string): boolean {
  return PHONE_REGEX.test(value)
}
```

## 10. 70/30 Ranking Formula (Pure Function)

```typescript
// @witness [RM-001]
// lib/utils/ranking.ts

interface RankingInput {
  dqs_score: number      // 0.0 – 1.0
  distance_km: number    // actual distance
  radius_km: number      // search radius
  w_quality?: number     // default 0.7 (from system_config)
  w_dist?: number        // default 0.3 (from system_config)
}

export function computeRankedScore({
  dqs_score,
  distance_km,
  radius_km,
  w_quality = 0.7,
  w_dist = 0.3,
}: RankingInput): number {
  const distanceComponent = 1 - Math.min(distance_km / radius_km, 1.0)
  return w_quality * dqs_score + w_dist * distanceComponent
}
```

This function is the testable counterpart to the Postgres `searching_nearby_profiles()` RPC. Unit tests must verify they produce identical results for the same inputs.
