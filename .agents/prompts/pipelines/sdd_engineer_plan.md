## 1. Database & RLS

### Migration file
Create `supabase/migrations/<timestamp>_<spec-id-slug>.sql`.

Every migration must:
- Enable RLS: `ALTER TABLE <name> ENABLE ROW LEVEL SECURITY;`
- Include GiST index for any `GEOGRAPHY` column: `CREATE INDEX ... ON ... USING GIST(location)`
- Include `@witness [SPEC-ID]` comment at the top

RLS policy syntax (from `docs/database/rls_policies.md`):
```sql
CREATE POLICY "<table>_<role>_<operation>"
  ON <table> FOR <SELECT|INSERT|UPDATE|DELETE>
  TO authenticated
  USING (<condition>)
  WITH CHECK (<condition>);   -- INSERT/UPDATE only
```

## 2. Backend — lib/ Structure

Follow this exact directory layout (defined in `docs/core/PATTERNS.md`):
```
app_build/
  lib/
    supabase/
      client.ts       ← browser client (use in Client Components only)
      server.ts       ← server client (use in Server Components + API routes)
    db/
      schema.ts       ← Kysely DatabaseSchema type (generated by kysely-codegen)
      index.ts        ← Kysely instance connected to Supabase
    queries/
      <feature>.ts    ← one file per domain (e.g., profiles.ts, connections.ts)
    types/
      index.ts        ← shared domain types
```

### Supabase server client (API routes and Server Components)
```typescript
// @witness [SPEC-ID]
// lib/supabase/server.ts
import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'

export function createClient() {
  const cookieStore = cookies()
  return createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    { cookies: { getAll: () => cookieStore.getAll(), setAll: (c) => c.forEach(({ name, value, options }) => cookieStore.set(name, value, options)) } }
  )
}
```

### Kysely instance
```typescript
// @witness [SPEC-ID]
// lib/db/index.ts
import { Kysely, PostgresDialect } from 'kysely'
import { createClient } from '@supabase/supabase-js'
import type { Database } from './schema'

// Kysely is used for typesafe queries only.
// For RLS-enforced queries, always pass user JWT via Supabase client.
export const db = new Kysely<Database>({ dialect: new PostgresDialect({ pool: ... }) })
```

### Typesafe query pattern (Kysely)
```typescript
// @witness [SPEC-ID]
// lib/queries/profiles.ts
import { db } from '@/lib/db'

export async function getProfileById(id: string) {
  return db
    .selectFrom('profiles')
    .select(['id', 'display_name', 'persona_type', 'city', 'state', 'dqs_score'])
    // Never select: phone_primary, phone_secondary, email_business — use get_visible_contact_info()
    .where('id', '=', id)
    .where('deleted_at', 'is', null)
    .executeTakeFirst()
}
```

### Calling a Postgres RPC from a Server Component
```typescript
// @witness [SPEC-ID]
// app/search/page.tsx (Server Component)
import { createClient } from '@/lib/supabase/server'

export default async function SearchPage() {
  const supabase = createClient()
  const { data, error } = await supabase.rpc('searching_nearby_profiles', {
    searcher_lat: 19.076,
    searcher_lng: 72.877,
    radius_km: 50,
    role_filter: null,
    page_size: 20,
    page_offset: 0,
  })
  // data is typed — see docs/api/API_CONTRACT.md §searching_nearby_profiles
}
```

### Atomic DB transaction (connections + address_book + audit)

## 3. API Routes

All routes live in `app_build/app/api/<resource>/route.ts`. Pattern:

```typescript
// @witness [SPEC-ID]
// app/api/profiles/[id]/route.ts
import { createClient } from '@/lib/supabase/server'
import { NextResponse } from 'next/server'

export async function GET(req: Request, { params }: { params: { id: string } }) {
  const supabase = createClient()
  const { data: { user }, error: authError } = await supabase.auth.getUser()
  if (!user) return NextResponse.json({ success: false, error: { code: 'UNAUTHENTICATED' } }, { status: 401 })

  // Non-PII fields: query directly
  const { data: profile } = await supabase
    .from('profiles')
    .select('id, display_name, persona_type, city, state, dqs_score')
    .eq('id', params.id)
    .single()

  // PII fields: MUST go through the handshake gate
  const { data: contact } = await supabase
    .rpc('get_visible_contact_info', { p_viewer_id: user.id, p_target_id: params.id })

  return NextResponse.json({ success: true, data: { ...profile, contact } })
}
```

Error response format (from `docs/api/API_CONTRACT.md`):
```typescript
return NextResponse.json(
  { success: false, error: { code: 'INSUFFICIENT_CREDITS', message: '...', details: {} } },
  { status: 402 }
)
```

## 4. Frontend Components

- File header: `// @witness [SPEC-ID]`
- All components are in `app_build/app/<route>/` or `app_build/components/`
- Use **Server Components** by default; add `'use client'` only for interactivity
- Styling: Tailwind only — use the tokens from `docs/core/STYLEGUIDE.md`
- Data fetching in Server Components: Supabase client directly (no `fetch`)
- Data fetching in Client Components: SWR with Supabase client
- Loading states: always include `<Suspense fallback={<Skeleton />}>`
- Error states: always include — never leave a component that can fail silently
